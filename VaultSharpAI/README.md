# VaultSharpAI

Кроссплатформенный десктопный и консольный клиент для работы с моделями искусственного интеллекта (LLM), построенный на базе современного стека **.NET 10.0** и **Avalonia UI**.

Проект разработан как универсальный независимый клиент для взаимодействия с любыми OpenAI-совместимыми API (DeepSeek, OpenRouter, ChatGPT, Ollama и др.) с гибкими возможностями настройки и управления контекстом, а также для демонстрации паттернов проектирования MVVM, DI и работы с мультимодальными запросами.

---

## Ключевые возможности

### Графический интерфейс (Desktop)
* **Управление профилями (API Profiles):** Переключение между сохраненными провайдерами (по умолчанию настроены OpenRouter и локальный Ollama). Автоматическое сохранение новых конфигураций Base URL/API Key.
* **Автоопределение моделей (Model Sync):** Загрузка актуального списка моделей напрямую с сервера по эндпоинту `/models` в один клик.
* **Режим Мышления (Thinking / Reasoning):** Полная поддержка вывода этапов размышления моделей (например, DeepSeek-R1). Шаги размышлений отображаются в блоке `🤔 Думаю:...` над основным ответом.
* **Режим Deep Research (Web Search):** Интеграция веб-поиска для провайдеров, поддерживающих эту опцию (передается флаг `web_search: true`).
* **Гибкая настройка токенов:** Точный лимит длины генерации (`max_tokens`) от 512 до 16384 с помощью регулятора в интерфейсе.
* **Прикрепление файлов (`📎`):**
  * **Изображения** (`.png`, `.jpg`, `.jpeg`, `.webp`) автоматически кодируются в `Base64` и передаются в запросе как структуры `image_url` (поддержка Vision-моделей).
  * **Текстовые файлы** считываются целиком и бесшовно внедряются в контекст сообщения в качестве пре-промпта.

### Консольная версия (Console CLI)
* Минималистичное и быстрое текстовое ядро для интерактивного диалога через терминал с поддержкой динамического подключения к любому API.

## Стек технологий

* **Язык программирования:** C# 14
* **Платформа:** .NET 10.0 SDK
* **Графическая библиотека:** [Avalonia UI 12.0.3](https://github.com/AvaloniaUI/Avalonia) (Кроссплатформенный WPF-подобный UI фреймворк)
* **Архитектурный паттерн:** MVVM (Model-View-ViewModel) на базе [CommunityToolkit.Mvvm 8.4.1](https://github.com/CommunityToolkit/dotnet)
* **Сеть и парсинг:** `HttpClient`, `System.Net.Http.Json`, `System.Text.Json`

## Требования и Запуск

### Системные требования
* **.NET 10.0 SDK** (или .NET 8.0 SDK)
* Операционная система: Windows 10/11, macOS или Linux (Ubuntu/Debian с установленным X11/Wayland)

### Сборка и запуск через CLI
Для запуска проекта склонируйте репозиторий и перейдите в его корень:

```bash
git clone https://github.com/irving2019/VaultSharpAI.git
cd VaultSharpAI
```

#### Запуск десктопной версии:
```bash
dotnet run --project VaultSharpAI.Desktop/VaultSharpAI.Desktop.csproj
```

#### Запуск консольной версии:
```bash
dotnet run --project VaultSharpAI/VaultSharpAI.csproj
```

## Инструкция по использованию (Desktop UI)

### Шаг 1: Настройка подключения и выбор модели
1. В левой боковой панели выберите готовый профиль из списка (**OpenRouter** или **Локальная модель**) либо введите свои данные вручную:
   * **Base URL:** Например, `https://api.deepseek.com/v1/` или `https://api.openai.com/v1/`
   * **API Key:** Ваш секретный токен доступа к API.
2. Нажмите кнопку **"Синхронизировать"**. Лампочка в шапке покажет зеленый индикатор **online** при успешном соединении, либо красный **offline** при ошибке.
3. Начните вводить название модели в поле ввода **"Модель ИИ"** — список автоматически отфильтрует модели по первым буквам совпадения. Выберите нужную модель.

### Шаг 2: Выбор параметров генерации
* **Мышление (Thinking):** Рекомендуется включать для моделей-рассуждателей (например, семейства DeepSeek R1). Это активирует разметку процесса мышления.
* **Deep Research:** Включает встроенный поиск информации в сети (доступно для моделей, поддерживающих веб-поиск через API).
* **Макс. токенов:** Укажите лимит длины генерации с помощью регулятора в боковой панели.

### Шаг 3: Общение и файлы
* Введите текст вашего вопроса в нижнее поле ввода и нажмите клавишу **Enter** (или круглую кнопку отправки **▲**).
* **Прикрепление файлов (`📎`):** Нажмите на скрепку в левом нижнем углу.
  * Если выбран текстовый файл, появится зеленый бейдж с его именем, а содержимое автоматически отправится вместе с запросом в виде контекста.
  * Если выбрано изображение, оно передастся в модель для распознавания (поддерживается на Vision моделях).
